## 1. 개요

농협사료 사무관리직 임직원을 위한 「생성형 AI 기반 사무관리 업무혁신 실무」 1일 8차시 과정의
학습사이트를 처음부터 끝까지 구축·배포했다. 기존 **bulid**(건설기계) 프로젝트를 베이스로
2권×6일 구조를 **단일 과정 × 8차시** 로 재편하고, 농협(NH) 아이덴티티로 리브랜딩했다.

## 2. 기술 스택

- React 18 · Vite 5 · React Router 6 (HashRouter)
- Tailwind CSS (Pretendard, Font Awesome 6)
- Supabase — 구글·카카오 OAuth, 학습 진도 동기화(`nonghyupsaryo_progress`, RLS)
- GitHub Actions → GitHub Pages 자동배포 (커스텀 도메인, base `'/'`)

## 4. 브랜딩 (농협 CI)

- 색상: **딥 그린 #12522e + 골드 #fec20d** — 골드값은 제공받은 공식 CI 파일에서 실제 색 추출

## 6. 배포 이슈 & 해결

- **빈 화면(중요)**: 신규 Pages가 `legacy`로 시작 → 리포 루트의 소스 `index.html`(→`/src/main.jsx`)을 서빙.
  → `gh api -X PUT .../pages -f build_type=workflow` 전환 **후 워크플로 재실행**으로 dist 아티팩트 활성화. 해결.
- **로그인 문제**: 로그인 자체는 정상(카카오 세션 확인). 원인은 `nonghyupsaryo_progress` **테이블 부재**로 진도 저장 실패.
  → 코드 방어처리(try/catch) + 테이블 수동 생성(`supabase/nonghyupsaryo_progress.sql`)으로 해결. 테이블 200 확인.
